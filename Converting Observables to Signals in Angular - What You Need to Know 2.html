Angular v16 introduces the new package rxjs-interop, which comes with a handy function called toObservable that allows developers to convert a signal to an observable. In this article, we‚Äôll explore this new feature and how it can be used.

First, we need to import the toObservable function from the @angular/core/rxjs-interop module. Here‚Äôs an example code snippet that demonstrates its usage:

import { toObservable } from '@angular/core/rxjs-interop';
import { signal } from '@angular/core';
@Component({
  standalone: true,
  template: `
    <input (input)="value.set(input.value)" #input />
  `,
})
export class FooComponent {
  value = signal('');
  value$ = toObservable(this.value);
  ngOnInit() {
    this.value$.subscribe(console.log);
  }
}

We create a signal with an initial value of an empty string, and then we use the toObservable function to convert it into an observable. We then subscribe to the observable and log its value every time it changes.

Under the hood, the toObservable function creates a ReplaySubject and wraps the provided signal in an effect. When the signal value changes, the effect will emit the new value.

One benefit of using signals is that we don‚Äôt need to add the distinctUntilChanged operator, as the work is already done at the signal level. Additionally, if the signal throws an error, it will be passed as an error notification to the subscriber.

One interesting effect of using the effect (üòÑ) to emit values is that the value will be delivered asynchronously. This means that even if we emit multiple values at the same ‚Äútick‚Äù, we‚Äôll only receive the last signal value.

It‚Äôs important to note that when the toObservable function is called, it checks to ensure that it is being called in an injection context. If not, an error will be thrown. This means that we can use the toObservable function only when the inject() function is available, except for cases when we pass an injector explicitly.

@Component({
  standalone: true,
  template: `
    <input (input)="value.set(input.value)" #input />
  `,
})
export class FooComponent {
  value = signal('');
  private injector = inject(Injector);
 
  ngOnInit() {
    const value$ = toObservable(this.value, {
      injector: this.injector
    }).subscribe(console.log);
  }
}

This is because Angular will complete the underlying ReplaySubject and destroy the effect registration when the wrapping context is destroyed. It does this using the new OnDestroy hook that it obtains from using the inject() function.
